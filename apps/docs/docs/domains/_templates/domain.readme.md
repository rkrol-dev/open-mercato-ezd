Użyj tego szablonu tworząc nową domenę/pod‑domenę.

## Minimalny zestaw plików
- overview.md (cel, zakres, definicje granic)
- sources.md (akty prawne, wytyczne, interpretacje)
- glossary.md (pojęcia)
- processes.md (procesy i zdarzenia)
- requirements/business.md (wymagania biznesowe)
- requirements/legal.md (wymagania formalnoprawne)
- mapping/open-mercato.md (mapowanie + luki)

## Standard ID dla wymagań
- `<SUBDOMAIN>-BIZ-###` i `<SUBDOMAIN>-LEG-###`
Przykład: `AP-LEG-001`.
